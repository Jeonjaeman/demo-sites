# MOA 데모 — 기획·설계 노트

요청사항([`../요청사항.txt`](../요청사항.txt)) 분석 후, 제안용 데모로 **무엇을 어떻게 보여줄지**를 정리한 문서입니다.

## 1. 핵심 판단

요청은 **3개의 사용자군**(일반 유저 · 관리자 · 광고주)이 **하나의 백엔드(Supabase)** 위에서 동작하는 구조다.
데모의 목표는 "이 팀이 ① 사용자 게시판 UX, ② 백오피스 운영, ③ 자체 광고 트래킹을 모두 이해하고 만들 수 있다"를 한눈에 보이는 것.
→ 랜딩 1 + 3개 화면(app/admin/advertiser)으로 구성하고, **차별점인 자체 광고/트래킹**을 가장 도드라지게 시연한다.

## 2. 디자인 방향

- 기존 데모(운송포탈=네이비 물류, 창업지원협회=라이트 공공)와 **의도적으로 다른 톤**.
- 모던 **소셜앱** 무드 → 바이올렛/인디고(`#6d5cf5→#a855f7`) + 코랄 액센트(`#ff5d8f`), 영문 디스플레이 `Space Grotesk`, 한글 `Pretendard`.
- 사용자 게시판은 **폰 목업(WebView)** 안에 렌더링해 "앱 속 웹뷰" 컨셉을 직관적으로 전달.

## 3. 화면별 시연 포인트

### app.html — 사용자 커뮤니티 (웹뷰)
- 피드: 공지/투표/일반 글 + 인스트림 광고(피드 카드형) + 하단 슬라이드형 캐러셀.
- 인터랙션: 좋아요·스크랩 토글, 투표(결과 바), 스포일러(탭하여 공개), **AI 번역**(로딩→번역→원문 보기 토글), 신고(사유 선택), 댓글/대댓글 상세 시트, 글쓰기 컴포저(제목·500자 카운터·이미지 3장·클라이언트 압축 안내·스포일러/투표 옵션).
- 탭: 전체/인기/투표/공지/내 글/스크랩.
- 광고 5종 전부 시연: 피드 카드형, **전면 팝업형**(진입 인터스티셜), **하단 배너형**(고정), **바텀시트형**(피드 끝 도달 시), 하단 슬라이드형.
- **클릭 → 리다이렉트 인터스티셜**로 외부 링크 라우팅을 시연.
- 우측 **실시간 트래킹 로그 패널**: 광고가 보이면 노출, 탭하면 클릭 이벤트가 Batch 큐에 쌓이는 흐름을 노출/클릭/CTR 카운터로 시각화.

### admin.html — 관리자 백오피스 (7 패널)
- 대시보드: 노출/클릭/CTR/미처리 신고 KPI + 7일 노출·클릭 듀얼 차트 + 진행 캠페인/최근 신고.
- 회원·권한: **4단계 권한** 승인/반려(권한 신청 큐) + 권한 변경 + **제재(정지/해제)** + 검색/필터 + CSV.
- 콘텐츠: 공지/광고/일반 목록 + **HTML 에디터** + **예약 발행**(일시 지정) + 임시저장.
- 신고: 접수/처리완료 큐 + 숨김/삭제/제재/반려 처리.
- 캠페인: 카드형 목록(상태·집행률·예산) + **새 캠페인 모달**(광고주·5종 타입·기간·타겟·예산, 자동 만료 안내) + 일시중지.
- 광고 소재: 5종 규격 프리뷰 그리드 + 운영 중 소재 성과 테이블.
- 트래킹: 7일 듀얼 차트 + 캠페인별 성과 + **CSV 내보내기**(Excel 호환 BOM, 실제 다운로드).

### advertiser.html — 광고주 대시보드
- 로그인 광고주(프레시밀 키트) 본인 데이터만: KPI(노출/클릭/CTR/집행금액) + 7일 추이 + 내 캠페인 + 소재별 성과(CTR 바) + **리포트 CSV**.

## 4. 비기능 요구의 데모 표현

| 요구 | 데모에서의 표현 |
|------|------------------|
| 노출/클릭 Batch 트래킹(DB 부하↓) | 우측 트래킹 로그 + "5초/100건 Batch 적재" 코멘트 |
| AI 번역 API Key 비노출(Edge Function 프록시) | 번역 로딩 문구 "Edge Function 프록시" + DB 미저장 명시 |
| 캠페인 자동 만료 | 종료일 경과 시 자동 만료 안내 + 기간 진행률 바 |
| SEO 메타(Slug/OG/JSON-LD) | 랜딩 `<meta og:*>` + 기능 카드에 명시 |
| 이미지 클라이언트 리사이징/압축 | 컴포저 이미지 추가 시 "리사이징·압축 완료" 토스트 |

## 5. 실제 구축 시 대체 지점

- `assets/js/data.js`의 목 데이터 → Supabase 테이블/RPC/Realtime.
- 트래킹 카운터 → `impressions`/`clicks` 이벤트 테이블 + Edge Function Batch insert.
- 번역 → Edge Function 프록시(번역 API Key 서버 보관, 클라 캐시).
- 백오피스 UI → 상용 템플릿(MUI 등) 커스터마이징(예산 절감), 사용자 게시판은 자체 디자인.

## 6. 비고

- 의존성 없는 순수 Vanilla(HTML/CSS/JS) — 빌드 과정 없음, GitHub Pages 정적 배포.
- 다른 데모와 **독립 폴더**(자체 `assets/`)로 상호 간섭 없음.
- 모든 인물/브랜드/수치는 가상 데이터.
