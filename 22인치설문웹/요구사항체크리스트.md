## 4. 추가 보강 제안 (납품 협의 항목)

1. **키오스크 모드 운영 스크립트** — 크롬 `--kiosk --incognito` 실행 배치파일 제공, 윈도우 시작 시 자동 실행 (현장 스태프가 더블클릭만 하면 되도록)
2. **중복 응답 정책** — 동일인 반복 응답 허용 여부 결정 필요 (행사장 특성상 보통 허용, 필요 시 세션 간격 제한)
3. **사은품 응모 연계** — 추첨 이벤트를 걸 경우 연락처 수집 → 개인정보 수집·이용 동의 절차와 파기 정책이 추가로 필요
4. **화면 번인 방지** — 장시간 동일 화면 노출 대비 대기 화면에 미세 모션 적용(데모 반영됨), OLED 패널이면 주기적 픽셀 시프트 권장
5. **다국어(영문) 토글** — 해외 관람객 비중이 있다면 문항·특장점 이중 언어 구성
6. **날짜별 필터/집계** — 행사가 여러 날이면 관리자 화면에 일자별 필터·엑셀 시트 분리
7. **문항 관리(CMS)** — 질문·선택지·특장점 텍스트를 코드 수정 없이 바꾸는 간이 관리 기능 (클라이언트가 콘텐츠를 늦게 확정하는 리스크 대비)

## 5. 데모 ↔ 납품 갭 (정직한 고지)

- 저장소: localStorage → **웹 호스팅 DB(MySQL 등) + API** 교체 필요
- 엑셀: CSV → **.xlsx** (문항별 시트 분리) 업그레이드 가능
- 실시간: 폴링 → WebSocket/SSE 권장 (다수 키오스크 동시 운영 시)
- 관리자 인증: 데모 하드코딩 → 서버 세션 인증
