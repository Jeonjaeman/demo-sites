# 레퍼런스 정리 — 22인치 터치 설문 키오스크

요구사항 문서의 4개 화면(키오스크 1·2, 대시보드, 관리자)별로 참고할 만한 레퍼런스를 정리.

## 1. 키오스크 터치 UI (공통 레이아웃)

| 레퍼런스 | 참고 포인트 |
|---|---|
| [Behance — kiosk ui touch screen 검색](https://www.behance.net/search/projects/kiosk%20ui%20%20touch%20screen) | 전시/이벤트용 키오스크 풀스크린 레이아웃, 대형 터치 버튼, 브랜드 컬러 적용 사례 다수 |
| [Dribbble — kiosk-ui 태그](https://dribbble.com/tags/kiosk-ui) | 카드형 선택지, 진행 표시(스텝 인디케이터), 완료 화면 모션 패턴 |
| [Dribbble — touch screen kiosks](https://dribbble.com/skills/touch%20screen%20kiosks) | 22인치급 대형 화면에서의 여백/타이포 스케일 감각 |
| [Interactive Kiosk Demo (oohfixer)](https://kiosk.oohfixer.com/) | 실제 브라우저에서 구동되는 키오스크 데모 — 화면 전환·아이들(대기) 화면 흐름 |

**적용 원칙**
- 터치 타깃 최소 80px 이상, 버튼 간격 넉넉하게 (장갑/서툰 터치 대응)
- 한 화면 = 한 가지 행동 (질문 1개 + 선택지만), 스크롤 금지
- 대기 화면(Attract screen) → 터치 시 시작 → 완료 후 자동 복귀 루프

## 2. 설문 진행 UX (키오스크 1 — 4문항 순차형)

| 레퍼런스 | 참고 포인트 |
|---|---|
| Typeform | "한 화면에 한 질문" 순차 진행의 원조 패턴. 진행률 표시, 부드러운 전환 |
| 맥도날드/롯데리아 키오스크 | 큰 카드형 선택 버튼, 하단 고정 이전/다음 내비게이션 |

**적용 원칙**
- 상단 진행률(1/4, 2/4…) 상시 노출 — 이탈 방지
- 선택 즉시 시각 피드백(선택 상태 강조) 후 다음 버튼 활성화
- 완료 화면: 감사 메시지 + N초 카운트다운 후 첫 화면 자동 복귀

## 3. 실시간 결과 노출 (키오스크 2 + 대시보드)

| 레퍼런스 | 참고 포인트 |
|---|---|
| [Mentimeter — Live Polling](https://www.mentimeter.com/features/live-polling) | 투표 직후 막대그래프가 실시간으로 차오르는 연출 — 키오스크 2의 "응답 직후 결과 노출"에 직접 대응 |
| [Slido](https://www.slido.com/) | 발표 화면용 대형 결과 차트(비율 % + 투표수 병기), 심플한 색 대비 |
| [Poll Everywhere — 2026 도구 비교](https://www.polleverywhere.com/use-cases/best-audience-response-tools-2026) | 실시간 응답 도구들의 결과 화면 유형(막대/도넛/워드클라우드) 개관 |

**적용 원칙**
- 결과는 막대그래프 + 비율(%) + 투표수 병기, 애니메이션으로 차오르게
- "내가 고른 항목" 강조 표시 → 참여 만족감
- 대시보드는 멀리서도 읽히도록 큰 수치(총 참여자 수) 중심 구성

## 4. 관리자 / 엑셀 다운로드

- 구글 폼 응답 시트, 네이버 폼 응답 관리 화면 수준의 단순한 표 + 다운로드 버튼이면 충분
- 엑셀 호환은 UTF-8 BOM CSV로 처리 (한글 깨짐 방지) — 실제 납품 시 xlsx 라이브러리 적용 가능

## 데모에 반영한 종합 방향

1. **키오스크 1** = Typeform식 순차 설문 + 맥도날드 키오스크식 대형 카드 버튼
2. **키오스크 2** = 제품 특장점 탭 UI + Mentimeter식 투표 직후 실시간 결과 연출
3. **대시보드** = Slido 발표 화면식 대형 차트 + 총계 수치
4. **관리자** = 응답 테이블 + 엑셀(CSV) 다운로드
