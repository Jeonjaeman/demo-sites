# 미식데스크 (MISIK DESK) — 타겟 키워드 정보 통합 관리 백오피스

맛집 관련 정보를 **한 곳에서 등록·검색·분류·협업·관리**하는 폐쇄형 내부 업무 시스템 **제안용 데모**.
RFP(타겟 키워드 자동 수집 및 백오피스 / AI 코딩 툴 활용)의 **1차 MVP 범위**를 중심으로 구현하고,
**2차 고도화(공개 정보 자동수집·중복 감지·알림·통계)** 는 미리보기로 포함했습니다.

> UI/UX는 RFP 선호(Notion · Slack · CRM 수준의 정돈된 화면)에 맞춘 라이트 어드민 + 다크 사이드바.

## 실행 방법
정적 사이트입니다. 레포 루트에서 로컬 서버를 띄워 여세요(CDN 폰트 때문에 `file://`보다 http 권장):

```bash
python -m http.server 8810
# http://127.0.0.1:8810/타겟키워드자동수집/index.html  (로그인 → 백오피스)
```

## 화면 구성
| 파일 | 내용 |
|------|------|
| `index.html` | **로그인** — 폐쇄형 시스템 안내 + 권한별 데모 계정(관리자/매니저/일반) 원클릭 입장 |
| `app.html` | **백오피스 SPA 셸** — 사이드바·통합검색·상세 드로어·모달·토스트 (해시 라우팅) |
| `assets/css/style.css` | 자체 디자인 시스템 (라이트 어드민 + 다크 사이드바) |
| `assets/js/data.js` | 목(mock) 데이터 + 인메모리 스토어(localStorage 영속) — **ERD/API 설계 출발점** |
| `assets/js/app.js` | 라우터 + 전 화면 렌더 + 상호작용(필터/검색/상세/협업/자동수집/CSV) |

## RFP 요구사항 ↔ 데모 매핑

### 1차 MVP (구현)
| RFP 항목 | 구현 위치 |
|----------|-----------|
| 로그인 / 사용자·권한 (폐쇄형, 권한 차등) | `index.html` 로그인 · 관리자 `#/users` · 권한별 메뉴/버튼 가드 |
| 정보 등록·수정·삭제·복구, 등록/수정자·일시 자동기록 | `#/info` 등록 모달 · 상세 드로어 · 변경 이력 타임라인 |
| 첨부파일(이미지/PDF/문서) | 상세 드로어 첨부 영역(데모 표시) |
| 통합 검색 + 다중 조건 필터 | `#/info` 검색창 + 카테고리·상태·중요도·검증·출처·지역 필터 |
| 카테고리/상태 관리(중요도·처리·출처·검증) | `#/taxonomy` (관리자) + 상세 드로어 인라인 변경 |
| 협업(내부 메모·댓글·담당자·이력) | 상세 드로어 협업 섹션 |
| 즐겨찾기 / 아카이브(보관 분리) | `#/favorites` · `#/archive` · 행/드로어 토글 |
| 관리자 대시보드(현황·통계·최근·추이) | `#/dashboard` KPI·카테고리 현황·키워드 추이·최근 등록 |
| 활동 로그(접속·작업 이력) | `#/logs` |

### 2차 고도화 (미리보기)
| RFP 항목 | 구현 위치 |
|----------|-----------|
| 키워드 기반 공개 정보 연동 / 자동수집 | `#/collect` — 타겟 키워드 → 외부 검색 API 조회(목업) |
| URL·유사 데이터 중복 등록 방지 | `#/collect` 결과의 **중복 감지** 표시 + 수집→정보 등록 |
| 알림(긴급/키워드/이메일·텔레그램·Slack) | `#/settings` 알림 조건 |
| 통계·급상승 키워드 | `#/dashboard` 급상승 키워드 |
| AI 기반 분류·요약 | 자동수집 등록 시 요약 자동 생성(데모 수준) |

## 데모 사용 팁
- 로그인 화면에서 **권한별 계정**을 바꿔 입장하면 권한 차등(메뉴/삭제/관리 버튼)이 동작합니다.
- 정보 행 클릭 → **상세 드로어**에서 상태·중요도·검증·담당자 변경, 메모/댓글, 이력 확인.
- `자동수집` → **즉시 수집 실행** → 중복 감지/신규 구분 → **정보로 등록**.
- 변경 사항은 브라우저 localStorage에 저장됩니다. `설정 → 데모 데이터 초기화`로 리셋.

## 실제 구축 시 (RFP 반영)
- **스택(예시)**: FE React/Next.js · BE Node/Python · DB PostgreSQL · BaaS Supabase/Firebase 활용, AI 코딩 도구(Cursor/Copilot) 기반 바이브코딩
- **공개 정보 연동**: Google Custom Search API 등 **합법적 공개 데이터/검색 API** + 약관 준수, 자동 수집 불가 항목은 수동 등록 병행
- **보안**: 권한 기반 접근제어, 민감정보 암호화, 접속/작업 로그, 백업·복구
- **산출물**: 소스코드 원본 · DB 설계서 · 관리자 매뉴얼 · 운영/배포 가이드

> 본 데모의 데이터는 목(mock)이며 실제 수집/업체 정보가 아닙니다.
