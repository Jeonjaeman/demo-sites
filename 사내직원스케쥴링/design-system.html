01Overview

개요

이 페이지는 피그마를 대신하는 라이브 코드 디자인 시스템입니다. 실제 assets/style.css를 그대로 로드해 렌더링하므로, 여기서 보는 값은 곧 프로덕션에서 쓰이는 값입니다. 우상단의 밀도·테마 토글로 모든 예시가 실시간으로 바뀌는 것을 확인할 수 있습니다.

1. 현장 우선 (Field-First)

근거 — 다수 사용자가 장갑 낀 손, 직사광선 아래 옥외 현장에서 모바일로 접속합니다. 큰 터치 타깃(44px 이상)과 고대비, 단순한 정보 구조를 최우선으로 설계합니다.

2. 데이터 밀도와 가독성의 균형

근거 — 스케줄러는 수백 명의 기술자·수십 개 프로젝트를 한 화면에서 다뤄야 합니다. compact 밀도 모드와 tabular-nums, 명확한 위계의 타이포 스케일로 밀도와 가독성을 동시에 확보합니다.

3. 상태는 색+아이콘+텍스트 3중 표현

근거 — 충돌·가동률·휴가 승인 상태는 업무상 중대한 신호입니다. 색상만으로 전달하면 색각 이상 사용자가 배제되므로, 배지에는 항상 아이콘과 텍스트 라벨을 병행합니다(색맹 대응).

4. 모든 액션은 되돌릴 수 있게

근거 — 배정 삭제, 휴가 반려 등은 실무에 즉각 영향을 미칩니다. 파괴적 액션에는 확인 모달과 실행취소(undo) 토스트를 기본 제공해 실수 비용을 낮춥니다.

02Color Tokens

컬러 토큰

모든 값은 getComputedStyle로 브라우저가 실제 계산한 색을 실시간으로 읽어옵니다. 테마를 전환하면 아래 값과 대비 판정 뱃지가 즉시 재계산됩니다. 대비 판정은 WCAG AA 기준(본문 4.5:1 · 굵은 대형 텍스트 3:1 · 비텍스트 UI 요소 3:1)을 따릅니다.

본 시스템은 --accent(인터랙티브 강조: 링크·포커스·선택)와 --primary(솔리드 CTA, 중성 다크톤)를 분리합니다. 다수의 상태색(ok/warn/danger/info)이 공존하는 데이터 밀집 화면에서 브랜드 블루를 인터랙션 신호로만 남겨 정보 위계를 명확히 하기 위함입니다.

Surface & Ink

09Accessibility

접근성 체크리스트

포커스 링

모든 인터랙티브 요소는 :focus-visible에서 2px 아웃라인이 표시됩니다 — outline:2px solid var(--focus).

최소 터치 타깃 44px

현장 모바일 뷰의 주요 액션 버튼·탭바 아이템은 44×44px 이상을 확보합니다.

aria-label 필수

텍스트가 없는 아이콘 전용 버튼(.iconbtn, .btn--icon)에는 항상 aria-label을 명시합니다.

색상 단독 의존 금지

상태 표현은 색 + 아이콘 + 텍스트 라벨 3중 구조를 원칙으로 합니다(원칙 3 참고).

prefers-reduced-motion 대응

모션을 줄이도록 설정한 사용자에게는 모든 애니메이션·트랜지션이 0.001ms로 즉시 처리됩니다.

키보드 내비게이션

모달·드로어는 Escape로 닫히고, 열릴 때 첫 입력 요소로 포커스가 이동합니다. 탭 순서는 시각적 순서와 일치합니다.

10Handoff

개발 인계 노트

토큰을 CSS 변수로 그대로 사용

모든 색·간격·그림자는 :root의 CSS 커스텀 프로퍼티입니다. 새 컴포넌트를 만들 때도 하드코딩된 hex 값 대신 반드시 var(--token)을 사용하세요. 예: background:var(--surface); color:var(--ink);

다크모드 구현 방식

html[data-theme="dark"] 속성 선택자로 토큰 값 자체를 덮어씁니다. 컴포넌트 CSS는 테마 분기 없이 var(--token)만 참조하므로 다크모드 전용 스타일을 추가로 작성할 필요가 없습니다. 토글은 UI.applyTheme('dark'|'light') 호출 한 줄입니다.

밀도(Density) 구현 방식

html[data-density="compact"]가 --row-h, --cell-w, --tbl-row, --pad 등 레이아웃 치수 토큰만 재정의합니다. 색상 토큰과 완전히 분리되어 있어 테마 × 밀도 4가지 조합이 모두 안전하게 작동합니다.

컴포넌트 명명 규칙 (BEM 유사)

  • .block — 컴포넌트 루트 (예: .kpi, .card, .modal)
  • .block__element — 하위 요소 (예: .kpi__v, .card__hd)
  • .block--modifier — 변형 (예: .btn--primary, .badge--danger)
  • .is-* — 상태 클래스 (예: .is-on, .is-sel, .is-err)

Figma 대응표 (선택)

Figma로 역이관 시 스타일 이름 제안:

  • Surface/Ink → Color/Surface/*, Color/Text/*
  • Semantic → Color/Status/Ok|Warn|Danger|Info
  • Categorical → Color/Project/01~12
  • Radius/Shadow → Effect/Elevation-1~4, Radius/xs~full

스크립트 의존성

컴포넌트 동작(모달/드로어/토스트/팝오버/포맷터)은 assets/ui.js의 window.UI 네임스페이스로 제공됩니다. 목업 데이터는 assets/data.js의 window.DB에서 결정론적 시드로 생성됩니다.
