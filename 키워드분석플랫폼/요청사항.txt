[프로젝트 개요]
- 내부 광고 제안 및 결과 분석에 활용하기 위한 주요 포털 및 4대 SNS 매체 대상 키워드 수집 관리자 웹 구축

[프로젝트 배경 및 목표]
- 기존 작성된 네이버 검색 데이터 중심의 기획안을 전면 수정하여, 소셜 미디어 데이터를 중심으로 분석하는 시스템을 구축하고자 합니다. 내부 마케팅 인력이 광고 제안서를 작성하거나 결과를 분석할 때, 특정 키워드에 대한 네이버 검색량뿐만 아니라 인스타그램, 엑스, 유튜브, 틱톡에서의 언급량과 반응 데이터를 통합하여 조회할 필요가 있습니다.
  이를 위해 다수의 키워드를 동시 검색하고, 크롤링 및 API를 통해 수집된 데이터를 직관적인 그래프로 시각화하며, 해당 데이터를 엑셀로 추출할 수 있는 내부 전용 어드민 시스템을 개발하는 것이 최종 목표입니다.

[과업 범위]
1. 수행 범위
- 상세 기획: 1차 요구사항을 바탕으로 크롤링 방식 및 비동기 데이터 처리 로직을 포함한 상세 화면 설계
- UI/UX 디자인: 내부 관리자가 직관적으로 데이터를 확인할 수 있는 대시보드 형태의 인터페이스 디자인
- 프런트엔드/Client 개발: 조회된 데이터를 다양한 차트 라이브러리를 활용하여 시각화하는 웹 화면 개발
- 백엔드 개발: 매체별 수집 봇 개발, 공식 API 연동, 수집 대기열 처리 및 데이터베이스 구축
- 서버/DB/인프라 구성: IP 차단 방지를 위한 프록시 연동 및 크롤링 서버 환경 구축

2. 상세 기능 요구 사항

2-1. 키워드 검색 및 네이버 데이터랩 연동
- 키워드 다중 검색: 5개에서 최대 10개의 키워드를 한 번에 입력하여 동시 검색을 요청하는 기능
- 최초 검색일 조회: 해당 키워드가 시스템에서 최초로 검색되어 데이터가 수집된 일자 표기 기능
- 최근 30일 검색량 비교: 입력된 키워드 간의 최근 30일 누적 네이버 검색량 비교 차트 제공
- 네이버 연동: 네이버 데이터랩 API 및 네이버 검색광고 API를 활용한 기본 검색량 추출 기능

2-2. 4대 SNS 매체 언급량 데이터 수집
- 인스타그램 데이터 수집: 해시태그 기반 최근 게시물 수 및 언급량 추이 데이터 수집 기능
- 엑스 데이터 수집: 특정 검색어 기반 트윗 언급 횟수 및 추이 데이터 수집 기능
- 유튜브 데이터 수집: 영상 제목 및 설명 기준 검색어 포함 영상 수, 조회수 추이 수집 기능
- 틱톡 데이터 수집: 특정 해시태그 기준 관련 영상 수 및 언급량 데이터 수집 기능
- 수집 방식 적용: 공식 API 활용을 우선 검토하고, 불가한 매체는 크롤링 봇 개발 또는 외부 데이터 수집 서비스 연동 적용
- 비동기 수집 처리: 검색 요청 시 대기 화면을 노출하거나 백그라운드로 수집 후 완료 상태를 표시하는 로직 적용

2-3. 데이터 분석 대시보드 시각화
- 검색량 추이 그래프: 조회 기간을 설정하고 성별, 연령별 필터링을 적용하여 선형 차트 및 막대 차트로 노출하는 기능
- 월별 및 요일별 검색 비율: 설정된 기간 내 월별, 요일별 데이터 분포를 원형 차트로 노출하는 기능
- 성별 및 연령별 검색 비율: 연동 가능한 인구통계학적 데이터를 기반으로 연령대별, 성별 비율 차트 노출 기능
- 연관 키워드 조회: 검색 키워드와 연관된 파생 키워드 목록을 텍스트 클라우드 또는 그룹화하여 시각화하는 기능
- 엑셀 다운로드: 현재 화면에 조회된 모든 차트 수치 및 원천 데이터를 엑셀 파일 형식으로 저장하는 기능
