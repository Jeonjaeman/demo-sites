# 키워드펄스 (KEYWORD PULSE) — 네이버 + 4대 SNS 키워드 분석 BI 대시보드

광고 제안·결과 분석을 위한 **내부 전용 키워드 분석 어드민** 데모.
네이버 검색량과 **인스타그램·X·유튜브·틱톡** 언급량을 통합 수집·시각화하고 엑셀로 내려받습니다.
RFP(네이버 및 SNS 연동 키워드 분석 플랫폼 / 웹 데이터 분석·BI)의 화면·기능 골격을 구현한 **제안용 데모**입니다.

> 참고 서비스(키워드사운드·블랙키위) 톤을 다크 데이터 대시보드로 재구성. 채널 브랜드 컬러로 멀티채널 비교를 직관화.

## 실행 방법
정적 사이트입니다. 레포 루트에서 로컬 서버를 띄워 여세요:

```bash
python -m http.server 8810
# http://127.0.0.1:8810/키워드분석플랫폼/index.html
```

## 화면 구성
| 파일 | 내용 |
|------|------|
| `index.html` | 대시보드 — 키워드 다중검색·필터, 비동기 수집 오버레이, KPI, 5개 분석 탭 |
| `assets/css/style.css` | 다크 데이터 대시보드 디자인 시스템 |
| `assets/js/data.js` | 키워드별 결정적(seeded) 목 데이터 생성기 (네이버·4SNS·추이·인구통계·연관) |
| `assets/js/app.js` | 검색·비동기수집 플로우 + SVG 차트(라인/바/도넛/스파크) + 탭 + 엑셀 내보내기 |

## 의뢰서(요구사항 정의서) 0~6 ↔ 데모 매핑
> 상세 사양은 [`분석/요구사항정의서_전략기획팀.md`](분석/요구사항정의서_전략기획팀.md) 참고.

| 의뢰서 화면 | 데모 구현 |
|---|---|
| 0. 검색 페이지(최대 10개 동시) | 상단 키워드 칩 입력 + 분석 실행 + 비동기 수집 오버레이 |
| 1. 키워드 최초 검색일 | **검색량 비교** 탭 테이블(최초검색일, 2016-01-01 수집시작 클램프) |
| 2. 최근 한 달 검색량 | **검색량 비교** 탭 테이블(합산·정렬) + 비교 바차트 |
| 3. 검색량 그래프 | **검색량 추이** 탭 — 일/월/연 단위, 기간·성별·연령 필터, **마우스오버 툴팁**, 원본 데이터 테이블 |
| 4. 월별/요일별 검색 비율 | **월별·요일별** 탭 도넛(비율 소수 1자리) |
| 5. 연관 키워드 | **연관 키워드** 탭 — **클러스터 그룹** + 총검색량 + 정렬 + **클릭 시 해당 키워드 분석 이동** |
| 6. 성별/연령별 검색 비율 | **성별·연령별** 탭 — 성별 분할(검색량+비율) + 연령 막대(검색량+비율) |
| 공통: 엑셀 다운로드 | 상단 ‘엑셀 다운로드’ — 검색량·추이·월별/요일별·성별/연령·연관 전 영역 CSV(검색수+비율) |
| 공통: 검색량 정수 / 비율 소수1 / PC+모바일 합산 | 전 화면 포맷 적용 |
| 공고 확장: 4대 SNS 언급량 | **SNS 언급량** 탭(인스타·X·유튜브·틱톡) — 비동기 수집 |

## 실제 구축 시 (RFP 반영 / 제안 포인트)
- **수집 방식 명시 필요**(RFP 특이사항): 실시간 크롤링 / 사전 배치 수집 / 외부 API 유료 연동 중 택. 본 제안은 **공식 API 우선 + 미지원 매체 배치 크롤링/외부 솔루션 연동** 권장.
- **인프라**: IP 차단 방지 프록시, 수집 대기열(큐), 크롤링 서버 — 개발비와 **별도 분리 산정**(프록시/외부 솔루션 구독비).
- **네이버**: 데이터랩 API + 검색광고 API(검색량·연관어·성별/연령). SNS: 공식 API(YouTube Data API 등) 우선, 불가 매체는 크롤링/외부 데이터.
- **차트**: 본 데모는 무의존 SVG. 실제는 ECharts/Recharts 등 커스터마이징 가능.

> 본 데모의 데이터는 목(mock)이며 실제 수집 결과가 아닙니다.
