### 숨은 니즈 (문서 행간 + 벤치마킹에서 도출)
- **콜드스타트 해결이 진짜 문제**: AI 자동답변은 기능이 아니라 "빈 게시판 문제"의 해법. → 데모에서 AI 답변을 가장 눈에 띄는 위치(히어로, 사이드, 게시판 뱃지)에 배치.
- **리뷰 신뢰성 장치가 없으면 리뷰 공간은 실패**: 잡플래닛(승인 심사+기브앤테이크), Glassdoor(기브투겟) 검증 모델. → 데모에 "리뷰 1건 작성 시 전체 열람" 잠금 UI 구현.
- **명예훼손 법적 리스크**: 회사 리뷰 플랫폼은 정보통신망법 44조의2 임시조치 요청을 반드시 받게 됨. → 리뷰/게시글 상태값(공개·심사중·임시비공개)을 데이터 모델에 반영, 관리자 신고 큐에 "블라인드(임시비공개)" 처리 구현.
- **AI 답변 고지·면책**: AI 오답이 커뮤니티 신뢰를 깎지 않도록 "베타" 라벨 + 면책 문구 + "선배 답변 유도" 문구를 모든 AI 답변에 부착.
- **영업 특화는 별점 축에서 나온다**: RepVue처럼 일반 5축이 아닌 영업 지표 중심 축(급여·인센티브 / 워라밸 / 영업문화 / 성장·커리어 / **목표·평가 합리성**)으로 재정의.
- **채용 확장 대비 데이터 구조**: 회사(companies)를 리뷰·게시글·미래 채용공고가 모두 참조하는 마스터 엔티티로 설계. 회사 상세 페이지가 곧 미래의 채용 페이지 골격.

## 2. 디자인 방향 — catch.co.kr 디자인 시스템 카피

캐치 실사이트에서 실측(computed style 추출)한 토큰을 그대로 사용:

| 토큰 | 값 | 캐치에서의 근거 |
|------|-----|----------------|
| 서체 | Noto Sans KR | body font-family 실측 |
| 잉크 | `#272727` (보조 `#474747`/`#666`/`#999`) | 본문/보조 텍스트 실측 |
| 포인트 레드 | `#F45757` | 핵심 CTA("로그인하고 추천받기") 실측 |
| 소프트 배경 | `#F3F6F8`, `#F4F4F4`, `#EEF3FF` | 칩/하이라이트 배경 실측 |
| 라인 | `#E2E8EC`, `#F0F0F0` | 보더 실측 |
| 버튼 | 다크 필(`#272727`, r30) + 레드 필(r50) | "더보기"/CTA 버튼 실측 |
| 라운딩 | 카드 8~12px, 필 30px, 태그 4px | 실측 |
| 카피 톤 | "지금, ○○이에요!" 큐레이션형 화법 | 섹션 헤딩 실측 |

화이트 베이스 + 낮은 채도 배경 + 레드 원포인트 = 캐치의 "정보 밀도 높지만 가벼운" 채용 플랫폼 무드를 커뮤니티에 이식.

## 3. 사이트 구성 (IA)

```
index.html (사용자, SPA 라우팅)
├─ 홈 ─ 히어로(AI 실시간 타이핑 데모) + 실시간 인기글(5) + 오늘 가장 많이 본 회사(4)
│        + 오늘의 노하우 밴드 + 이번 주 실적 랭킹(3)   ← 전부 관리자 큐레이션 연동
├─ 커뮤니티 ─ 전체/자유/질문답변/실적인증/노하우 탭 + 사이드(AI 안내·인기글·리뷰 급상승)
│   └─ 게시글 상세 ─ 본문·좋아요·스크랩·신고 / AI 답변 블록(로딩→등록) / 댓글(BEST 정렬)
├─ 회사리뷰 ─ 검색 + 업종 필터 + 별점순 카드(영업 특화 축 미리보기)
│   └─ 회사 상세 ─ 평균 별점 + 5축 통계 바 + 익명 리뷰 리스트(기브앤테이크 잠금)
└─ 마이페이지 ─ 레벨/포인트/연속출석 + 뱃지 신청(증빙 업로드) + 활동 내역 + 프로필

admin.html (관리자, 6패널)
├─ 대시보드 ─ KPI 4종 + 주간 활동 차트 + AI 답변 현황(토큰 비용 리포트)
├─ 회원 관리 ─ 뱃지 신청 큐(증빙 확인→승인/반려) + 전체 회원 + 제재(정지/해제)
├─ 회사 관리 ─ 수기 등록 + 엑셀 일괄 업로드(파싱→중복 병합 시뮬레이션) + 크롤러 로드맵 고지
├─ 게시글 모니터링 ─ 목록 + 블라인드 토글
├─ 신고 처리 ─ 신고 큐(블라인드/기각) + 임시조치 정책 고지
└─ 메인 큐레이션 ─ 인기글 슬롯 순서 변경 + 오늘의 회사/노하우 교체
```

## 4. 화면별 시연 포인트

### 사용자 (index.html)
- **AI 자동답변 (핵심 차별점)**: 질문답변 게시판에 글 등록 → "AI가 답변 생성 중" 로딩 → 3초 후(실서비스 1~2분) 키워드 매칭 답변 자동 등록. 콜드콜/이직·연봉/미팅/인센티브 4개 주제 사전과 폴백 답변. 모든 AI 답변에 GPT 연동·베타 라벨 + 면책 문구.
- **회사 리뷰**: 관리자 등록 회사만 선택 가능(요구사항 그대로), 영업 특화 5축 별점 입력, 익명 보장 고지, 현직/전직 구분, 장단점 분리 입력. 회사 상세에 축별 평균 바 통계.
- **기브앤테이크 잠금**: 두 번째 리뷰부터 블러 + "리뷰 쓰고 전체 열람" — 리뷰를 쓰면 즉시 해제(localStorage 지속).
- **신뢰 장치**: 닉네임 옆 등급 뱃지 상시 노출(블라인드식 반익명), 실적인증 글 "증빙인증" 태그 + 자동 블러 고지.
- **체류 장치**: 레벨/포인트(+10P 댓글, +20P 글, +50P 리뷰), 연속 출석 스트릭.
- **반응형**: 860px 이하에서 모바일 하단 탭바 + 단일 컬럼, 어드민은 상단 가로 스크롤 내비로 전환.

### 관리자 (admin.html)
- 뱃지 큐 처리 시 사이드 카운터 실시간 감소, 엑셀 업로드는 진행 바 → "17행 파싱, 신규 15/중복 병합 2" 결과까지 시뮬레이션.
- 큐레이션 순서 변경(↑↓)이 데모 상태에 즉시 반영 — "메인 수동 배치" 요구사항의 조작감을 보여줌.
- AI 토큰 사용량/비용 위젯 — "토큰 비용 클라이언트 부담" 계약 조항에 대응하는 운영 리포트 제안.

## 5. 데모의 백엔드 표현 (요구: "확인 가능한 정도")
- 사용자 측은 **localStorage 를 DB 삼아** 글/댓글/리뷰/좋아요/열람권/뱃지신청이 새로고침 후에도 유지.
- 관리자 측은 인메모리 상태(새로고침 시 초기화)로 시연 반복이 쉽게.
- 실서비스 제안 스택(문서 명기용): React(Next.js) + Node.js(NestJS) + PostgreSQL + OpenAI API, 비밀번호 bcrypt, 이메일 인증(SES 등). 데모 화면 내 문구(비밀번호 암호화, 이메일 인증 발송, 임시조치 30일)로 비기능 요구 대응을 표기.

## 6. AI 자동답변 비용 통제 전략

"질문마다 무제한 GPT 호출"은 트래픽이 늘수록 토큰 비용이 선형 증가한다(비용은 클라이언트 부담 계약).
아래 5중 가드레일로 **비용 상한을 계약 가능한 수준으로 고정**한다. 데모 관리자 대시보드의 "AI 자동답변 현황" 위젯에 반영되어 있다.

| 층 | 장치 | 효과 |
|---|------|------|
| ① 호출 차단 | 사용자당 일일 질문 한도(기본 3회, 등급별 차등) + 최소 글자수·금칙어·중복 질문 필터 | 스팸·저품질 호출 원천 차단 |
| ② 캐시 재활용 | 질문 임베딩 유사도 검색 → 기존 AI/커뮤니티 BEST 답변이 있으면 API 호출 없이 재노출("이런 답변이 이미 있어요") | 반복 질문 비용 0원 + 축적 콘텐츠 재활용(SEO 부수효과) |
| ③ 호출 최적화 | 경량 모델 기본(gpt-4o-mini급) + `max_tokens` 상한 + 질문 길이 제한 + 배치 큐 처리 | 건당 단가 최소화 (상위 모델은 "전문가 검증 요청" 등 프리미엄 동선에만) |
| ④ 예산 상한 | 월 예산 상한(예: $50) 설정 → 80% 도달 시 관리자 알림, 100% 소진 시 AI 답변 자동 일시중지 + "선배들의 답변을 기다려보세요" 폴백 UI | 청구서 상한 보장 — 서비스는 계속 동작 |
| ⑤ 가시성 | 관리자 대시보드에 일/월 토큰 사용량·비용 환산·캐시 적중률 리포트 | 클라이언트가 비용을 직접 모니터링·조정 |

운영 전환 옵션: 트래픽이 안정화되면 "모든 질문 자동 답변" → "AI 답변 요청 버튼(옵트인)"으로 전환해
콜드스타트 해소 이후의 상시 비용을 더 낮출 수 있다.
