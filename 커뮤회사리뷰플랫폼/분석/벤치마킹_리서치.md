# 벤치마킹 리서치
**영업인 특화 커뮤니티 + 회사 별점·익명 리뷰 플랫폼 구축 프로젝트 (향후 채용 플랫폼 확장)**
리서치 기준일: 2026-07 / 방법: 웹 검색 기반 사실 확인(확인 불가 항목은 "추정" 표기)

## 1. 국내 (5)

### 1.1 블라인드(Blind) — 회사 이메일 인증 기반 익명 직장인 커뮤니티의 국내 표준
- **서비스 개요**: 회사 이메일로 재직을 인증해야 활동 가능한 익명 직장인 플랫폼. [2025년 3월 전 세계 가입자 1,200만 돌파](https://platum.kr/archives/254551), 국내 10대 그룹사 재직자 10명 중 9명 이상이 가입, 한국 가입자 연평균 소득 6,710만 원(직장인 평균 대비 약 60% 높음).
- **커뮤니티/리뷰 UX**: 완전 익명이되 닉네임 옆에 **회사명 배지**가 붙는 반(半)식별 구조가 신뢰의 핵심. 라운지(전체)/토픽(주제별)/회사 게시판 3층 구조. 커뮤니티와 별도로 [기업 리뷰·연봉 정보](https://www.teamblind.com/kr/company) 섹션을 운영 — 전·현직자의 별점과 장단점 리뷰를 커뮤니티 여론과 함께 보여준다.
- **신뢰성/모더레이션**: 이메일 인증 정보와 계정을 분리 저장해 "운영사도 작성자를 특정 못 한다"는 익명성 보장을 마케팅 전면에 내세움(추정: 단방향 암호화 구조). 신고 누적 기반 제재.
- **참고 포인트**: ① "익명 + 소속 배지" 문법 — 우리 뱃지(대표인증/전문가/영업인)를 닉네임 옆 배지로 상시 노출하면 같은 신뢰 효과. ② 커뮤니티 여론과 기업 리뷰를 한 회사 페이지에서 묶어 보여주는 구조. ③ 재직 인증(이메일/명함) 자체가 콘텐츠 신뢰 장치라는 점 — 증빙 업로드 승인 플로우의 근거.

### 1.2 잡플래닛(Jobplanet) — 리뷰 승인 심사 + 기브앤테이크 열람의 국내 원조
- **서비스 개요**: 국내 대표 기업 리뷰 플랫폼. 기업별 별점·리뷰·연봉·면접 후기 제공, 리뷰 데이터를 기반으로 채용 서비스로 확장한 선례.
- **커뮤니티/리뷰 UX**: 별점은 총점 + **5개 항목(승진 기회 및 가능성 / 복지 및 급여 / 업무와 삶의 균형 / 사내문화 / 경영진)**으로 구성. 장점·단점·경영진에게 바라는 점을 분리 입력받아 리뷰 품질을 구조적으로 확보.
- **신뢰성/모더레이션**: 작성된 리뷰는 [운영정책](https://www.jobplanet.co.kr/welcome/review_operation_policy)에 따라 **승인 심사를 거쳐야 공개**되고, 리뷰 1건이 승인되면 그날부터 일정 기간(6개월~1년, 시기별 상이) 다른 기업 리뷰 열람권을 얻는 **기브앤테이크 모델**. 기업이 권리침해를 소명하면 해당 리뷰를 '임시 비공개' 처리하는 [임시조치](https://www.jobplanet.co.kr/contents/news-508) 절차를 운영(정보통신망법 제44조의2 대응).
- **참고 포인트**: ① 리뷰 작성→관리자 승인→공개의 심사 큐를 백오피스에 그대로 이식. ② "리뷰 1건 쓰면 전체 열람 N개월" 열람권 모델 — 초기 리뷰 축적의 가장 검증된 장치. ③ 임시 비공개(블라인드) 상태값을 리뷰 데이터 모델에 처음부터 설계.

### 1.3 캐치(CATCH) — 기업정보 카드 UI와 콘텐츠 큐레이션 (본 프로젝트 디자인 레퍼런스)
- **서비스 개요**: [진학사가 운영하는 상위권 신입·경력 채용 플랫폼](https://www.catchapply.co.kr/). 대기업·중견 기업정보/채용공고/현직자 리뷰/캐치TV(영상 콘텐츠)를 결합한 취준생 대상 서비스.
- **커뮤니티/리뷰 UX**: 기업 페이지가 [기업요약](https://www.catch.co.kr/Comp/CompSummary/090096)·[재무평가](https://www.catch.co.kr/Comp/CompInfo/090096)·[현직자리뷰](https://www.catch.co.kr/Comp/ReviewInfo/090096)·인터뷰 탭으로 모듈화. 리뷰는 조직문화/워라밸/연봉수준/복지 항목 평가에 "직원의 81%가 추천", "전체 업종 상위 11%" 같은 **백분위 요약 지표**로 가공되어 카드 한 장으로 읽힘.
- **신뢰성/모더레이션**: 재무제표 기반 재무평가(상위 % 표시)처럼 주관 리뷰와 객관 데이터를 병렬 배치해 신뢰를 보완. 리뷰 검수 정책 상세는 비공개(추정: 내부 검수).
- **참고 포인트**: ① 회사 상세를 "요약 카드 + 탭(정보/리뷰/공고)" 구조로 — 우리 회사 페이지 레이아웃의 직접 레퍼런스. ② "직원 N% 추천·상위 N%" 요약 배지로 별점을 한 줄 요약. ③ 메인의 큐레이션 섹션(추천 기업/인기 콘텐츠 배치) — 관리자 메인 큐레이션 기능의 UI 참고.

### 1.4 리멤버 — 실명 DB 위의 익명 커뮤니티, 직무 기반 게시판의 정석
- **서비스 개요**: 명함 관리 앱에서 출발해 채용·커뮤니티·리서치로 확장, [2025년 2월 누적 회원 500만 돌파](https://www.unicornfactory.co.kr/article/2025022009553180352)(국내 화이트칼라 약 절반), 누적 스카웃 제안 800만 건.
- **커뮤니티/리뷰 UX**: [커뮤니티](https://community.rememberapp.co.kr/main)는 **직무 단위(인사/HR, 영업, 마케팅 등)로 방이 나뉘고**, 글·댓글에 작성자의 직무가 함께 표시됨. 실명 명함 DB 기반이지만 커뮤니티에서는 [닉네임/익명 선택](https://support.rememberapp.co.kr/hc/ko/articles/33946898223641)이 가능 — "실명 인증 + 익명 발화"의 균형이 성공 요인으로 꼽힘.
- **신뢰성/모더레이션**: 명함(직장·직함) 인증이 곧 자격 검증. [커뮤니티 가이드라인](https://support.rememberapp.co.kr/hc/ko/articles/900000184403) 기반 신고·제재.
- **참고 포인트**: ① 영업 직군만 따로 모으는 "직무 커뮤니티" 개념의 국내 실증 — 우리 서비스는 이를 통째로 버티컬화한 것이라고 포지셔닝 가능. ② 닉네임 옆 직무/등급 표시로 익명이어도 발언 무게를 부여. ③ 명함 업로드=증빙 승인 플로우(리멤버의 명함 OCR 승인)와 유사한 우리 뱃지 증빙 심사.

### 1.5 사람인 기업리뷰 (+원티드인사이트) — 대량 리뷰 축적과 공공데이터 결합의 두 갈래
- **서비스 개요**: [사람인 기업리뷰](https://www.saramin.co.kr/zf_user/company-review)는 전·현직자 리뷰 약 37.8만 건(2026-07 표기 기준)을 채용공고 옆에 붙여 놓은 채용 포털형 리뷰. 비교 대상인 [원티드인사이트](https://zdnet.co.kr/view/?no=20230622093942)(구 크레딧잡)는 국민연금·고용보험 공공데이터로 240만 개 기업의 연봉·입퇴사율을 리뷰 없이 보여주는 데이터형. 리뷰 UX 벤치마크로는 사람인이 더 유의미해 사람인을 주 분석 대상으로 선정.
- **커뮤니티/리뷰 UX**: 리뷰 항목은 **조직문화/경영진/근무환경/급여 및 복지/자기개발** 5개 축 + 통계 요약. 채용공고→기업정보→리뷰가 한 동선에 연결되어 "리뷰가 채용 전환을 돕는" 구조.
- **신뢰성/모더레이션**: 리뷰 열람에 로그인/작성 유도 장치를 두는 방식(추정: 시기별 정책 변동). 원티드인사이트처럼 공공데이터를 병렬 표기하면 익명 리뷰의 신뢰 보완 가능.
- **참고 포인트**: ① 리뷰 → 해당 회사 채용공고로 이어지는 동선 — 2단계 채용 확장 시 그대로 재현할 구조이므로 리뷰-회사-공고 테이블 관계를 미리 설계. ② 별점 5개 축을 우리 타겟에 맞게 재정의할 근거(아래 RepVue 참조). ③ 공공데이터(연봉·입퇴사율) 병기 아이디어 — 데모에서는 관리자 입력 필드로 흉내 가능.

## 2. 해외 (5)

### 2.1 Glassdoor — Give-to-Get 열람 모델과 리뷰 모더레이션의 세계 표준
- **서비스 개요**: 월 5,500만+ 방문(추정치 상이), [누적 리뷰 7,000만+ 건](https://expandedramblings.com/index.php/numbers-15-interesting-glassdoor-statistics/)의 세계 최대 기업 리뷰 플랫폼. 2021년 Fishbowl 인수 후 커뮤니티 기능을 통합했고, Indeed(리크루트홀딩스)와의 통합이 진행 중.
- **커뮤니티/리뷰 UX**: 기업 리뷰·연봉·면접·복지 4종 콘텐츠. 리뷰는 총점 + 세부 항목 별점, 장단점 분리 서술, CEO 지지율/추천율/전망 3지표.
- **신뢰성/모더레이션**: ["Give-to-Get" 정책](https://help.glassdoor.com/s/article/Give-to-get-policy?language=en_US) — 본인 경험 콘텐츠 1건을 제출하면 12개월 무제한 열람. [연구에 따르면](https://www.prnewswire.com/news-releases/new-study-reveals-glassdoor-give-to-get-policy-leads-to-more-balanced-company-ratings-300534547.html) 이 정책이 극단적 1점 리뷰를 3.6%p, 5점 리뷰를 2.1%p 줄이고 중간 평점을 늘려 **평점 분포 자체를 건강하게 만드는 효과**가 검증됨. [모더레이션](https://www.glassdoor.com/blog/the-why-behind-glassdoor-review-moderation-process/)은 기술 필터 1차 + 신고 시 인간 검수 2차, "콘텐츠를 절대 수정하지 않고, 부정적이라는 이유만으로 삭제하지 않는다"를 원칙으로 문서화.
- **참고 포인트**: ① 기브앤테이크를 "12개월 열람권"으로 명시한 규칙 설계. ② "수정하지 않는다/부정적이어서 지우지 않는다" 류의 **공개 모더레이션 원칙 문서** — 데모에도 리뷰 정책 페이지 1장을 포함하면 완성도가 급상승. ③ 자동 필터(금칙어·개인정보) + 신고 시 수동 검수의 2단 구조.

### 2.2 RepVue — 영업직 특화 회사 평가 플랫폼 (본 프로젝트와 가장 유사한 원형)
- **서비스 개요**: ["세계 최대 영업조직 평가 플랫폼"](https://www.repvue.com/). 수십만 명의 검증된 영업 직군 사용자가 남긴 [1,000만+ 데이터 포인트](https://www.repvue.com/data)로 회사별 영업조직 점수를 산출. 분기마다 [Cloud Sales Index](https://www.repvue.com/cloud-index/2025/Q2)(2025 Q2 기준 246개사, 약 4.7만 명 평가) 발행.
- **커뮤니티/리뷰 UX**: 평가 축이 일반 직장 리뷰와 완전히 다름 — **쿼타 달성률(연간 목표를 채우는 영업사원 비율 %), OTE(목표 달성 시 총보상), 기본급/인센 구조, 인바운드 리드 흐름, Product-Market Fit, 조직문화·리더십, 상위 퍼포머의 수입 상한** 등 영업인이 이직 시 실제로 따지는 지표만 별점화. 회사 페이지에 "이 회사 영업사원의 약 70%가 쿼타 달성" 같은 문장형 요약 제공.
- **신뢰성/모더레이션**: LinkedIn 연동 등으로 영업 직군 재직을 검증(추정: 세부 방식 비공개)한 사용자만 평가 가능. 평가 참여 시 상세 데이터 열람권을 주는 기브앤테이크형 접근.
- **참고 포인트**: ① 별점 항목을 잡플래닛식 5축이 아니라 **영업 특화 축(쿼타 달성률·인센티브 만족·리드 지원·영업문화·성장성)**으로 재정의 — 이 프로젝트의 핵심 차별화이므로 데모 리뷰 폼에 반드시 반영. ② "달성률 N%" 문장형 요약 지표. ③ 리뷰 데이터로 산업 리포트(인덱스)를 만드는 확장 로드맵 — 관리자 통계 화면의 근거.

### 2.3 Fishbowl — 3단계 익명 레벨 선택이라는 발명
- **서비스 개요**: 2021년 [Glassdoor가 인수](https://techcrunch.com/2021/09/14/glassdoor-fishbowl-linkedin/)한 반익명 직장인 커뮤니티. 업계/직무/회사 단위의 "Bowl"(그릇)에 가입해 대화하는 구조로, Glassdoor 커뮤니티 탭의 기반이 됨.
- **커뮤니티/리뷰 UX**: 게시할 때마다 [익명 수위를 3단계로 선택](https://techcrunch.com/2023/07/18/glassdoor-is-introducing-blind-like-anonymous-community-features-to-fuel-user-growth/) — (a) 회사명만 표시("Works at ○○"), (b) 직함만 표시("Sr. Account Executive"), (c) 실명. 글 성격에 따라 작성자가 노출 수준을 고르게 해, 완전 익명(블라인드식)과 실명(링크드인식) 사이의 스펙트럼을 확보.
- **신뢰성/모더레이션**: 회사 이메일/직장 검증 후 Bowl 입장 자격 부여. Glassdoor 통합 후 [계정·익명성 정책](https://help.glassdoor.com/s/article/Understanding-your-Glassdoor-and-Fishbowl-Accounts?language=en_US)을 공동 운영.
- **참고 포인트**: ① 글 작성 시 "익명 / 뱃지만 표시(영업인·전문가·대표) / 닉네임+회사" 노출 레벨 선택 UI — 우리 등급 뱃지 시스템과 결합하면 강력한 차별화. ② 게시판을 직무·업계 Bowl처럼 세분화(제약영업/B2B/보험/자동차 등 업종별 방). ③ 커뮤니티(Fishbowl)와 리뷰(Glassdoor)를 한 계정으로 잇는 통합 모델 — 우리 기획의 "커뮤니티+리뷰 단일 플랫폼"과 동일 방향.

### 2.4 Comparably — 설문형 리뷰 수집과 어워드 게이미피케이션
- **서비스 개요**: [기업문화·연봉 투명성 플랫폼](https://www.comparably.com/). 리뷰 데이터를 분기별 "Best Places to Work" 어워드로 재가공해 기업 브랜딩 상품으로 판매하는 B2B 수익모델이 특징.
- **커뮤니티/리뷰 UX**: 서술형 대신 **28개 평가 질문 + 7개 주관식**의 구조화 설문으로 리뷰를 수집, [16개 문화 지표](https://www.comparably.com/awards/rules)(보상/성장/리더십/워라밸/동료 등)로 점수화. 예/아니오·1~10점·객관식을 섞어 응답 부담을 낮추고 데이터 정형성을 높임.
- **신뢰성/모더레이션**: 최근 12개월 익명 응답만 집계, 어워드는 응모·비용 없이 **최소 참여 인원 기준(대기업 75명, 중소 25명)**을 채운 기업만 자동 선정 — 소수 리뷰로 순위가 왜곡되는 것을 방지.
- **참고 포인트**: ① 리뷰 작성을 "긴 글쓰기"가 아닌 "객관식 설문 + 짧은 주관식"으로 설계해 작성 전환율을 높이는 폼 디자인. ② "리뷰 N건 미만 회사는 통계 비공개" 최소 표본 규칙 — 초기 데이터가 적은 우리 서비스에 필수. ③ 연간 "영업하기 좋은 회사 TOP 10" 어워드 콘텐츠화 — 마케팅·B2B 확장 아이디어.

### 2.5 Reddit r/sales — 영업인 커뮤니티의 원형과 자치 모더레이션
- **서비스 개요**: [약 58.9만 멤버](https://redditagency.com/subreddits/sales)의 세계 최대 영업인 온라인 커뮤니티. 별도 서비스가 아닌 서브레딧이지만, "영업인끼리 모이면 무엇을 이야기하는가"를 보여주는 콘텐츠 원형.
- **커뮤니티 UX**: 콜드콜 스크립트, 쿼타 압박, 커미션 분쟁, 이직·연봉 협상, 번아웃 등 실전 주제가 주류 — [토론 대부분이 조언 요청형](https://gummysearch.com/r/sales/). 업/다운보트와 플레어(주제 태그)로 콘텐츠가 자율 정렬됨.
- **신뢰성/모더레이션**: 자기 서비스 홍보·리드 수집(prospecting) 엄금이 핵심 규칙(추정: 세부 규칙 문구는 서브레딧 내 게시). 레딧 전반의 [90/10 자기홍보 관행](https://www.onlinemoderation.com/market-on-reddit-without-getting-banned/)과 모더레이터 자치로 스팸을 통제 — 영업인 커뮤니티는 특성상 "회원끼리 서로에게 영업하는" 오염이 가장 큰 리스크임을 시사.
- **참고 포인트**: ① 게시판 카테고리를 r/sales 인기 주제로 역설계 — 자유(잡담·번아웃)/질문답변(콜드콜·협상 노하우)/실적인증(쿼타·커미션)/노하우(스크립트·툴)와 정확히 대응. ② "회원 간 영업·리크루팅 금지" 조항을 커뮤니티 규칙에 명문화(신고 사유 항목에 포함). ③ 플레어식 말머리 태그와 추천 정렬.

### 3-b. 의뢰자가 명시하지 않았지만 벤치마킹상 필수인 고려사항

1. **리뷰 승인 심사(선검수 후공개)**: 잡플래닛·Glassdoor 공통. 자동 필터(금칙어·실명·연락처) 1차 + 관리자 승인 2차. 데모 백오피스에 심사 큐 화면 필수.
2. **기브앤테이크 열람 모델**: 리뷰 1건 승인 시 N개월 전체 열람권(잡플래닛 6개월~1년, Glassdoor 12개월). 초기 리뷰 축적의 검증된 유일 장치이며, [평점 분포를 중앙으로 수렴시키는 효과](https://www.prnewswire.com/news-releases/new-study-reveals-glassdoor-give-to-get-policy-leads-to-more-balanced-company-ratings-300534547.html)까지 입증됨.
3. **명예훼손 법적 리스크 대응**: 기업 리뷰는 [정보통신망법상 명예훼손·임시조치 이슈](https://www.jobplanet.co.kr/contents/news-508)가 상수. 권리침해 신고 접수→30일 임시 비공개(제44조의2)→이의제기 플로우와 리뷰 상태값(공개/심사중/임시비공개/삭제)을 데이터 모델에 처음부터 반영, 리뷰 작성 폼에 "확인 가능한 사실 위주 작성" 가이드 고지.
4. **AI 답변 고지·면책**: AI 자동 답변에는 "AI가 생성한 참고용 답변" 라벨, 생성 시각, 오류 가능성 면책 문구를 고정 노출하고, 채택·좋아요 등으로 사람 답변과 구분되는 별도 스타일 적용(전문 자격이 필요한 조언으로 오인되지 않도록).
5. **익명성 보장 설계**: "유저에겐 익명, 시스템/관리자는 식별 가능"(블라인드·Fishbowl 계열의 정석). 닉네임 자동 생성 + 회사·실명 노출 금지, 단 백오피스에서는 제재·법적 대응을 위해 작성자 추적 가능해야 함. Fishbowl식 노출 레벨 선택(익명/뱃지만/닉네임)을 차별화 기능으로.
6. **영업 특화 별점 축 재정의**: 잡플래닛식 일반 5축 대신 RepVue처럼 인센티브·쿼타·리드 지원 등 영업인의 의사결정 변수를 축으로 — 이 프로젝트가 "영업인 특화"임을 증명하는 가장 가시적인 지점.
7. **최소 표본 규칙**: 리뷰 3건(가안) 미만 회사는 "통계 준비 중"으로 별점 비공개(Comparably의 최소 참여 기준 응용) — 리뷰 1건으로 회사 평판이 결정되는 왜곡과 작성자 특정(익명성 파괴) 위험을 동시 방지.
8. **회원 간 영업/리크루팅 금지 규칙**: 영업인 커뮤니티 최대 오염원은 회원끼리의 영업(r/sales의 핵심 금지 조항). 신고 사유에 "영업/홍보/리크루팅" 항목을 명시하고 반복 위반 시 등급 강등.
9. **채용 확장 대비 데이터 구조**: 회사(companies)를 리뷰·게시글·(미래의) 채용공고가 모두 참조하는 마스터 엔티티로 설계하고, 회사 페이지에 채용 탭 자리를 미리 확보 — 잡플래닛·RepVue가 걸은 리뷰→채용 전환 경로를 스키마 수준에서 준비.

**종합**: 국내외 공통 패턴은 (1) 리뷰 신뢰는 "재직 인증 + 선검수 승인 + 기브앤테이크"의 3종 세트로 만들어지고, (2) 익명 커뮤니티는 "익명이되 소속/자격 배지는 보이는" 반익명 구조가 승리 공식이며, (3) 영업 버티컬의 차별화는 RepVue가 증명했듯 별점 축 자체를 영업 지표(쿼타·인센티브)로 바꾸는 데서 나온다는 점입니다. 데모 단계에서는 리뷰 심사 큐(1), 기브앤테이크 열람 잠금(2), AI 답변 라벨(4), 영업 특화 별점(6)만 시연해도 "운영을 아는 기획"임을 보여줄 수 있습니다.
